About

The purpose of this project is to apply for the Python engineering job at Better Earth. I developed this project to demonstrate my skills in Python and showcase my ability to work on complex projects. It serves as a motivated tour de force, showcasing my dedication and passion for software development.

Difficulties Faced

  • One of the challenges I encountered was obtaining the token for Task 1. Due to the sensitive nature of the API, obtaining the token required additional steps and permissions. Although I was unable to acquire the token, this challenge allowed me to explore alternative solutions and enhance my adaptability.

  • Another difficulty arose when deploying the application on PythonAnywhere. The API for utility rates that I used was not widely supported by the platform. To overcome this, I actively engaged with the PythonAnywhere community by posting on the forum to request the addition of the API. This initiative showcased my determination and willingness to explore solutions beyond the initial roadblocks.

Technologies Used

For this project, I utilized a combination of HTML, CSS, and JavaScript to develop the website's frontend. HTML provided the structure and semantic markup for the webpages, while CSS was used for styling and visual enhancements. JavaScript was employed to add interactivity and dynamic features to the website.

In addition to these frontend technologies, I integrated external APIs to enhance the functionality of the application. I leveraged the utility rate API to retrieve utility rate data and the Google Geocoding API to convert addresses into geographic coordinates. For data storage, I utilized a MySQL database to store and retrieve information efficiently.

To handle the backend logic and serve the web application, I used Flask, a lightweight web framework in Python. Flask allowed me to create routes, handle requests, and interact with the database seamlessly. The combination of these technologies enabled me to build a robust and user-friendly web application.
