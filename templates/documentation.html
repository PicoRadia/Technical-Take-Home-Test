Introduction

Welcome to my the BetterEarth Technical take home exam Documentation! This documentation serves as a comprehensive guide on Task 3 of the technical exam.

By integrating the fire code lookup API from AHJ Registry and the utility rates database API from NREL's Utility Rates v3, we can provide a comprehensive tool for our design engineering team to assess fire pathways and determine the average kWh residential utility rate for a given location.

The documentation will showcase the Python Flask application and discuss the implementation details, including the use of OOP principles and the integration of the two APIs. We'll explain how the application takes an address input, retrieves the fire code and utility rate information, and presents the results in a JSON format.

Key Features

  • Login Portal

    The login portal allows users to securely access the application by authenticating their credentials. The login system is connected to a SQL database for user management and authentication.

  • Address Search and Results

    This functionality combines task 1 and task 2. Users can enter an address into the application and retrieve relevant results or information based on the address provided. The application utilizes geolocation services to process and retrieve data related to the given address.

  • Feedback Page

    The feedback page allows users to provide feedback, suggestions, or request additional functionalities. Users can submit messages, comments, or suggestions through the feedback form, and you can review and respond to their input accordingly.

Getting Started

To get started with the application, follow the steps below:

  1. Clone the repository to your local machine.
  2. Install the required dependencies using pip install -r requirements.txt.
  3. Configure the API keys for AHJ Registry and NREL's Utility Rates v3 in the appropriate configuration file.
  4. Run the Flask application using python flask_app.py.
  5. Access the application through the provided URL.
